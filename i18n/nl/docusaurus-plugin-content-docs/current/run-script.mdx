#### Structuur

```JavaScript

```

![1](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/1.png)

##### setFilters(reeks objecten, string)

Gebruikt om filters in te stellen.

**De eerste parameter** is een array van objecten. Elk object vertegenwoordigt een filtervoorwaarde. Het bevat drie kenmerken:

**`id`** : Dit verwijst naar de id van deze kolom. Als deze kolom een resultaat is van een database-query, dan zou het ID de kolomnaam zijn die u heeft opgehaald uit de query. Bijvoorbeeld, als de query `SELECT e-mail AS user_email FROM gebruikers` is en `email` als `E-mail` hernoem via het rechterpaneel van de tabel. het id van de kolom zou nog steeds `user_email` zijn als deze kolom handmatig wordt toegevoegd met behulp van de knop `Kolom toevoegen` in het rechterpaneel je kan het ID checken in het `displayedData` gedeelte van het linkerpaneel Meestal zal het een willekeurig gegenereerde string zijn die begint met `column` zoals `column-320169e8-225a-482d-9853-ad34b3040220`.

**`filterFn`** : het wedstrijdtype van het filter. We ondersteunen de volgende types: `equalTo` die hetzelfde is als `=` in SQL, `notEqualTo` welke hetzelfde is als `<>` in SQL, `contains` wat hetzelfde is als `LIKE '%value%'` in SQL, `does NotContain` wat hetzelfde is als `NIET LIKE '%value%'` in SQL, `lessThan` is hetzelfde als `<` in SQL, `notLessThan` dat hetzelfde is als `>=` in SQL, `notMoreThan` is hetzelfde als `<=` in SQL, `empty` dat hetzelfde is als `IS NULL` in SQL, `notEmpty` welke hetzelfde is als `IS NIET NULL` in SQL, `voor` wordt gebruikt om tijd of datum te vergelijken, wat hetzelfde is als `>` in SQL, `after` wordt gebruikt om tijd of datum te vergelijken, wat hetzelfde is als `<` in SQL.

**`value`** : gebruik om de waarde te vergelijken.

**De tweede parameter:** wanneer er meerdere filtervoorwaarde objecten zijn, bepaalt het of ze een "AND" relatie of een "OR" relatie hebben. Zet het als een string `and` voor een "AND" relatie, en `of` voor een "OF" relatie.

###### Ondersteunde componenten

Tabel

###### Voorbeeld

```JavaScript
table1.setFilters([{"id":"maand", "filterFn":"equalTo", "value":"May"}, {"id":"maand", "filterFn":"equalTo", "value":"June"}, "or")
```

### Ga naar URL

#### Structuur

```JavaScript
utils.goToURL({
url:string,
newTab:boolean
})
```

#### Beschrijving

Gebruik een object om het in te stellen. Er zijn 2 attributen in het object. Het eerste attribuut is `url`. De waarde is een volledige URL, inclusief 'https://' of 'http://', en het is van het type string. Het tweede attribuut is `newTab`. Het is een boolean die gebruikt wordt om de URL te openen in een nieuw tabblad.

#### Voorbeeld

```JavaScript
utils.goToURL({
url:"https://www.illacloud.com",
newTab:true
})
```

### Opslaan op ILLA Drive

#### Structuur

```JavaScript
utils.saveToILLADrive({
fileName:string,
fileData:string,
fileType:string,
folder:string,
allowAnonymous:boolean,
replace:boolean
})
```

#### Beschrijving

Wordt gebruikt om een bestand op te slaan op de ILLA Drive.

`fileName`: Naam met de extensie van het bestand na uploaden naar ILLA Drive.

`fileData`: Base64 of raw file data, zoals JSON, CSV, platte tekst, enz.

`fileType`: Een van de volgende waarden: Auto, Plain text, JPEG, PNG, SVG, JSON, CSV, TSV, Excel(.xlsx). Bij gebruik van "Auto," zullen we proberen het bestandstype te bepalen op basis van ofwel de bestandsextensie of de bestandsgegevens.

`folder`: Doelmap voor bestandsupload. Laat leeg om te uploaden naar de hoofdmap.

`allowAnonymous`: Wordt gebruikt om in te stellen of anonieme gebruikerstoegang is toegestaan. Als u van plan bent uw applicatie als publiek in te stellen, moet anonieme toegang worden ingeschakeld. Voor de beveiliging van uw gegevens moeten bestanden of mappen anoniem toegankelijk worden gemaakt in de anonieme map.

![13](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/13.png)

`replace`: Wordt gebruikt om in te stellen wanneer er bestanden met dezelfde naam in dezelfde map staan, vervang het bestaande bestand door een nieuw bestand of hernoem het nieuwe bestand automatisch.

#### Voorbeeld

```JavaScript
//gebruik upload1 om bestanden van het apparaat te kiezen en te uploaden naar de app-map. 
hulp.saveToILLADrive({
fileName:upload1.files[0].name,
fileData:upload1. iles[0].dataURI,
fileType:"PNG",
map:"apps",
allowAnonymous:false,
replace:true
})
```

```JavaScript
/upload een tekstbestand.  
utils.saveToILLADrive({
fileName:"test.txt",
fileData:"abc",
fileType:"Plain text",
folder:"apps",
allowAnonymous:false,
replace:true
})
```

#### Algemene gegevens instellen in

##### Structuur

```JavaScript
utils.setGlobalDataIn({
key:string, 
path:string, 
value:any
})
```

##### Beschrijving

Wanneer de globale data een array of object is, gebruik setGlobalDataIn om de waarde te wijzigen die overeenkomt met een specifieke index in een matrix of om de waarde te wijzigen naar een specifieke sleutel in een object.

`key`: de weergavenaam van globalData.

`path`: de index van de waarde die je wilt bijwerken in de array of de sleutel in het object. Bijvoorbeeld, voor het instellen van de tweede waarde in array, is het pad 1 (de index). Om de waarde in te stellen van het attribuut met de naam key1, is het pad sleutel1.

`value`: de waarde om in te stellen

##### Voorbeeld

```JavaScript
//objectState is een object: {"key1":"value1", "key2":"value2"}, om de 
utils.setGlobalDataIn({
key:"objectState", 
path:"key2", 
value:input7.value
})
```

utils.downloadFromILLADrive({
