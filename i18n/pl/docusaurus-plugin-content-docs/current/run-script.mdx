## ILLA custom functions

### Wyzwalacz

#### Struktura

```JavaScript
const exampleData = oczekiwanie actionName.trigger();
```

Nazwa akcji jest ciągiem wyświetlanym na zrzucie ekranu, w następujący sposób.

![1](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/1.png)

#### Opis

W Uruchom skrypt zmienna `exampleData` jest taka sama jak `actionName.data`.

#### Przykład

Na przykład używamy akcji `postgresql1` do zapytania wiadomości e-mail James.

![2](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/2.png)

Wywołaj akcję poprzez uruchomienie skryptu

![3](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/3.png)

`postgresql1.data` jest

```JSON
[
  {
    "email": "James@outlook.com",
    "id": "1",
    "img": "https://images.pexels.com/photos/1222271/pexels-photo-1222271.jpeg?auto=compress&cs=tinysrgb&w=1600",
    "nazwa": "James"
  }
]
```

Kiedy musimy użyć JavaScript do pobrania wiadomości e-mail w kontroli komponentu lub gdzie indziej, możemy użyć następujących informacji: `{{postgresql1.data[0].email}}`

![4](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/4.png)

##### setFilters(tablica obiektów, ciąg)

Używane do ustawiania filtrów.

**Pierwszy parametr** jest szeregiem obiektów. Każdy obiekt reprezentuje warunek filtrowania. Obejmuje on trzy cechy:

**`id`** : Odnosi się to do id tej kolumny. Jeśli ta kolumna jest wynikiem zapytania do bazy danych, to id będzie nazwą kolumny, którą pobrałeś z zapytania. Na przykład, jeśli zapytanie jest `SELECT email AS user_email od użytkowników` i zmień nazwę `email` na `Email` przy użyciu panelu prawej ręki tabeli, id kolumny będzie nadal `user_email` Jeśli ta kolumna zostanie ręcznie dodana za pomocą przycisku `Add Column` w panelu prawostronnym, możesz sprawdzić jego identyfikator w sekcji `displayedData` w lewym panelu. Zazwyczaj będzie to losowo wygenerowany ciąg znaków zaczynający się od `column` takiego jak `column-320169e8-225a-482d-9853-ad34b3040220`.

**`filterFn`** : typ dopasowania filtra. Obsługujemy następujące typy: `equalTo`, który jest taki sam jak `=` w SQL, `notEqualTo` który jest taki sam jak `<>` w SQL, `contains` który jest taki sam jak `LIKE '%value%'` w SQL, `does NotContain` który jest taki sam jak `NOT LIKE '%value%'` w SQL, `lessThan`, który jest taki sam jak `<` w SQL, `notLessThan`, który jest taki sam jak `>=` w SQL, `notMoreThan` która jest taka sama jak `<=` w SQL, `empty` która jest taka sama jak `IS NULL` w SQL, `notEmpty` która jest taka sama jak `IS NOT NULL` w SQL, `before` jest używany do porównywania czasu lub daty, co jest takie samo jak `>` w SQL, `after` jest używany do porównywania czasu lub daty, co jest takie samo jak `<` w SQL.

**`wartość`** : używana do porównania wartości.

**Parametr drugi:**, gdy istnieje wiele obiektów warunkowych z filtrem, określa, czy mają one związek "AND" czy "OR". Ustaw jako ciąg `and` dla zależności "AND" i `or` dla zależności "OR".

###### Obsługiwane elementy

Tabela

###### Przykład

```JavaScript
table1.setFilters([{"id":"month", "filterFn":"equalTo", "value":"May"}, {"id":"month", "filterFn":"equalTo", "value":"June"}], "lub")
```

### Zapisz na dysku ILLA

#### Struktura

```JavaScript
utils.saveToILLADrive({
fileName:string,
fileData:string,
fileType:string,
folder:string,
allowAnonymous:boolean,
replace:boolean
})
```

#### Opis

Używane do zapisywania pliku na dysku ILLA.

`fileName`: Nazwa z rozszerzeniem pliku po przesłaniu na Dysk ILLA.

`fileData`: Base64 lub surowe dane z plików, takie jak JSON, CSV, zwykły tekst, itp.

`fileType`: Jedna z następujących wartości: Auto, Plain text, JPEG, PNG, SVG, JSON, CSV, TSV, Excel(.xlsx). Gdy używasz "Auto", spróbujemy określić typ pliku na podstawie rozszerzenia pliku lub danych pliku.

`folder`: Folder docelowy do przesyłania plików. Pozostaw puste, aby przesłać do katalogu głównego.

`allowAnonymous`: Używane do ustalania, czy dostęp anonimowy użytkownika jest dozwolony. Jeśli planujesz ustawić swoją aplikację jako publiczną, należy włączyć dostęp anonimowy. Dla bezpieczeństwa Twoich danych, pliki lub foldery dostępne anonimowo muszą być umieszczone w folderze anonimowym.

![13](https://cdn.illacloud.com/official-website/img/docs/new-doc/run-script/13.png)

`replace`: Używany do ustawiania plików o tej samej nazwie w tym samym katalogu, zastąp istniejący plik nowym plikiem lub automatycznie zmień nazwę nowego pliku.

#### Przykład

```JavaScript
//użyj upload1 aby wybrać pliki z urządzenia i przesłać je do folderu aplikacji. 
utils.saveToILLADrive({
fileName:upload1.files[0].name,
fileData:upload1. iles[0].dataURI,
fileType:"PNG",
folder:"apps",
allowAnonymous:false,
replace:true
})
```

```JavaScript
//wgraj plik tekstowy.  
utils.saveToILLADrive({
fileName:"test.txt",
fileData:"abc",
fileType:"Plain text",
folder:"apps",
allowAnonymous:false,
replace:true
})
```
